# Инструкция по публикации портфолио на GitHub Pages

Этот проект построен на базе **React + Vite + Tailwind CSS**. Ниже описано, как опубликовать этот сайт на GitHub Pages всего за несколько шагов.

---

## Шаг 1: Подготовка Git-репозитория

Если вы еще не инициализировали Git в проекте:

1. Создайте новый публичный репозиторий на вашем аккаунте GitHub (например, с именем `portfolio`).
2. В терминале на вашем локальном компьютере выполните следующие команды:

```bash
git init
git add .
git commit -m "feat: initial seasons portfolio commit"
git branch -M main
git remote add origin https://github.com/ВАШ_НИКНЕЙМ/portfolio.git
```

---

## Шаг 2: Установка пакета `gh-pages`

Пакет `gh-pages` автоматически собирает проект и отправляет скомпилированные файлы в специальную ветку `gh-pages`, с которой GitHub раздает статические сайты.

Запустите команду в терминале:

```bash
npm install --save-dev gh-pages
```

---

## Шаг 3: Настройка конфигурационных файлов

### 1. Настройка `vite.config.ts`

Чтобы Vite правильно разрешал пути к картинкам, скриптам и стилям на серверах GitHub, вам нужно указать параметр `base` равным имени вашего репозитория.

Откройте `vite.config.ts` и добавьте строку `base`:

```typescript
export default defineConfig({
  base: '/portfolio/', // Укажите имя вашего репозитория между косыми чертами!
  // остальные настройки...
})
```

*Примечание: Если ваш сайт будет открываться по основному домену (например, `https://username.github.io/` без имени папки в конце) или у вас подключен кастомный домен, укажите `base: '/'`.*

### 2. Настройка `package.json`

Добавьте поле `"homepage"` и скрипты развертывания.

Откройте `package.json` и добавьте сверху:
```json
"homepage": "https://ВАШ_НИКНЕЙМ.github.io/portfolio",
```

Затем в блок `"scripts"` добавьте две команды:
```json
"scripts": {
  "predeploy": "npm run build",
  "deploy": "gh-pages -d dist",
  // ваши старые скрипты...
}
```

---

## Шаг 4: Запуск публикации (Деплой)

Теперь все готово! Выполните в терминале простую команду:

```bash
npm run deploy
```

Эта команда автоматически:
1. Запустит сборку проекта (`npm run build`).
2. Создаст или обновит локальную ветку `gh-pages`.
3. Опубликует файлы из папки `dist` в ваш репозиторий на GitHub.

Сайт станет доступен по адресу:  
`https://ВАШ_НИКНЕЙМ.github.io/portfolio/`  
*(Публикация на стороне GitHub может занять от 1 до 5 минут при первом запуске).*

---

## Почему мы разработали портфолио на React (а не Vue)?

Поскольку среда разработки Google AI Studio полностью оптимизирована и предварительно настроена под **React 19, Tailwind CSS v4 и Framer Motion**, мы разработали этот сайт на React для обеспечения максимальной скорости рендеринга интерактивных холстов (Canvas), плавной прокрутки и поддержки современных библиотек. 

Однако, если вы хотите перенести этот проект на Vue.js:
1. Вы можете переиспользовать все стили Tailwind CSS (они абсолютно идентичны).
2. Вы можете перенести логику анимации частиц в компонент Vue `<canvas>` в хук `onMounted()`, используя тот же самый рендеринг и формулы физики.
3. Код шаблонов структурирован максимально понятно для переноса!
